🎙️ Agentic AI Flow (ReAct Pattern)

Autonomous reasoning loop: Observe → Think → Act → Evaluate → Retry

🤖 True Agentic Behavior — Not Just API Calls

GARUDA AI implements a ReAct (Reasoning + Acting) agent loop. The agent autonomously decides what SQL to generate, executes it as a tool, evaluates whether results are sufficient, and retries with corrected queries if the first attempt fails — up to 3 autonomous attempts before falling back.

1
Observe
Receive user query + conversation context
2
Think
Agent decides which tool to use (SQL gen)
3
Act
Generate SQL + Execute on DB
4
Evaluate
Check: rows > 0? Error? Relevant?
5
Retry / Fix
If failed → feed error back → regenerate SQL
6
Analyze
Criminological reasoning + recommendations

💡 Innovation & Creativity

Novel problem framing and creative use of AI agents

🎯 Novel Problem Framing

We don't just call an API — we built a self-correcting autonomous crime investigation agent that implements the ReAct (Reasoning + Acting) pattern: it generates SQL as a tool, executes it, evaluates results, and autonomously retries with corrected queries on failure. Combined with criminological reasoning through established sociological frameworks, this creates an AI partner that thinks like a detective.

🔄
ReAct Agent Loop (Genuine Agentic AI)
Not a simple prompt→response. The agent:
  • Observes: User query + conversation memory
  • Thinks: Decides which tool (SQL) to invoke
  • Acts: Generates & executes SQL on database
  • Evaluates: Checks if results are sufficient
  • Retries: Self-corrects errors up to 3 attempts
  • Reasons: Produces criminological analysis
Full reasoning trace visible to user for transparency.
🧠
AI as a Criminologist
Every response is grounded in criminological theory — not just "here's the data," but "here's why this pattern exists and what to do about it."

  • Agent Confidence scoring on every response

☁️ Google Tool Utilization

Depth of Gemini, Vertex AI, and Cloud Run integration

🧠
Gemini 1.5 Flash — 4-Agent Autonomous System
We use Gemini in a ReAct agent loop with 4 specialized roles:
  • Agent 1 — SQL Tool Agent: Schema-aware NL→SQL with self-correction. Retries on error with error context injection. temp=0.
  • Agent 2 — Result Evaluator: Autonomously decides if results are sufficient or need retry (genuine decision-making).
  • Agent 3 — Criminology Analyst: Receives full reasoning trace + results, generates theory-grounded insights with confidence scoring.
  • Agent 4 — MO Pattern Matcher: Cross-references active case against full DB for serial crime detection + prevention strategy.
⚡

  • Vertex AI SDK (google-cloud-aiplatform) for authenticated model access
  • Service Account auth — no API keys, IAM-based security
  • Cached model initialization for fast subsequent requests
  • Generation config tuning: temp=0 for SQL, default for analysis
  • Graceful fallback between model versions

🔑 Key Integration Insight

Unlike simple API-call demos, GARUDA AI implements a genuine ReAct agent loop: the agent autonomously generates SQL, executes it as a tool, evaluates results, self-corrects on errors (up to 3 retries), and feeds its full reasoning trace into the analysis agent. This demonstrates agentic AI with tool-use, autonomous decision-making, and self-correction — not just prompt engineering.
